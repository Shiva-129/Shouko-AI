1. REQUIRED ACCOUNTS
=====================

You must create accounts at the following services. I list each with why,
what plan you need, and the monthly cost.

1. Supabase (supabase.com)
   Why: Auth (JWT tokens, user management), realtime, database hosting
   Plan: Free tier (up to 50,000 monthly active users, 500 MB database)
   Cost: $0/month (free tier sufficient for beta)
   Note: You can also self-host Postgres + use a separate auth provider,
         but Supabase is the integrated auth solution the app is built for.

2. OpenRouter (openrouter.ai)
   Why: LLM API gateway — provides the embedding model (nvidia/llama-nemotron)
         and fallback for other LLMs. This is the PRIMARY AI provider.
   Plan: Pay-as-you-go. No monthly fee, usage-based billing.
   Cost: ~$5-20/month for 100 users (embeddings are cheap, ~$0.01/1M tokens)

3. Cloudflare (cloudflare.com)
   Why: R2 object storage for PDF files. The app stores uploaded PDFs in R2.
   Plan: Free tier (10 GB storage, 10 million reads/month, 1 million writes/month)
   Cost: $0/month (free tier sufficient for beta)
   Note: You only need R2 (S3-compatible object storage), not the full Cloudflare
         CDN/spectrum unless you also want to proxy through Cloudflare.

4. Resend (resend.com)
   Why: Transactional email — sends welcome emails and daily digest emails to users.
   Plan: Free tier (100 emails/day, 3,000 emails/month)
   Cost: $0/month (free tier sufficient for 100 users)
   Note: The app uses Resend SDK. The from-email defaults to onboarding@resend.dev
         on the free tier, or you can use a custom domain on the paid plan ($10/mo).

5. Stripe (stripe.com)
   Why: Payment processing — handles pro plan subscriptions, checkout sessions,
         billing portal, and webhook events.
   Plan: Pay-as-you-go. 2.9% + $0.30 per transaction.
   Cost: $0/month (no monthly fee, only transaction fees)
   Note: Stripe is OPTIONAL for beta launch. The app works without it — users
         stay on the free plan. Only set up Stripe when you're ready to charge.

6. Sentry (sentry.io)
   Why: Error tracking and performance monitoring.
   Plan: Free tier (5,000 events/month, 1 user)
   Cost: $0/month (free tier sufficient for beta)
   Note: OPTIONAL for launch. The app works without Sentry (graceful skip).

7. GitHub (github.com) — you already have this
   Why: Source control and CI/CD (GitHub Actions)
   Plan: Free tier (2,000 CI minutes/month)
   Cost: $0/month

8. Docker Hub (hub.docker.com) — only if publishing images
   Why: Container registry for production deployments
   Plan: Free tier (1 private repository, unlimited public)
   Cost: $0/month
   Note: You can build and deploy directly from source without Docker Hub.

=====================

  Root domain:     shouko-ai.app (or your choice)
  API subdomain:   api.shouko-ai.app
  App subdomain:   app.shouko-ai.app (or use root for the app)

  DNS Records needed:
    Type    Name              Value                              TTL
    ─────────────────────────────────────────────────────────────────
    A       api.shouko-ai.app  <VPS_IP_ADDRESS>                   300
    A       app.shouko-ai.app  <VPS_IP_ADDRESS>                   300
    (or CNAME records if using a platform like Railway/Vercel)

  Optional:
    CNAME   pdfs.shouko-ai.app <r2-bucket-url>.r2.cloudflarestorage.com   300
    (for custom R2 public URL — not needed if you use the default R2.dev URL)

  NOTE: For a private beta, you can skip custom domains entirely and use:
    Frontend: http://<VPS_IP>:3000
    API:      http://<VPS_IP>:8000
    You only need domains for a production-facing launch.

5. REQUIRED CLOUDFLARE SETUP
==============================

Only needed if using Cloudflare R2:

  1. Create R2 bucket
     Bucket name: shouko-pdfs (or whatever you set R2_BUCKET_NAME to)
     Location: auto (automatic)

  2. Generate R2 API token
     Permissions: Object Read & Write
     TTL: No expiration (or set reasonable limit)

  3. (Optional) Set up custom domain for R2
     In R2 → bucket → Settings → Public URL → Connect custom domain
     Domain: pdfs.shouko-ai.app
     This enables: https://pdfs.shouko-ai.app/paper-uuid/original.pdf

  Environment variables:
    R2_ACCOUNT_ID = Cloudflare Account ID (from Cloudflare dashboard)
    R2_ACCESS_KEY_ID = From the generated API token
    R2_SECRET_ACCESS_KEY = From the generated API token
    R2_BUCKET_NAME = shouko-pdfs
    R2_PUBLIC_URL = https://pub-xxxxx.r2.dev (from bucket settings)
                    OR https://pdfs.yourdomain.com if using custom domain

7. REQUIRED SUPABASE SETUP
=============================

  1. Create a new Supabase project
     Project name: shouko-ai
     Database password: (save this — it's part of DATABASE_URL)
     Region: Choose closest to your users

  2. Auth providers
     Email/Password: ENABLED by default — this is what the app uses
     The app signup/login pages use Supabase Auth UI

  3. Redirect URLs (for auth callbacks)
     Go to Authentication → URL Configuration:
     Site URL: http://localhost:3000 (dev) + https://app.shouko-ai.app (prod)
     Redirect URLs:
       http://localhost:3000/**
       https://app.shouko-ai.app/**

  4. SQL migrations
     The app handles schema via Alembic. You do NOT need to run Supabase
     migrations. Run from your backend:
       cd apps/api && alembic upgrade head
     This creates all tables: users, papers, paper_chunks, artifacts,
     conversations, annotations, collections, daily_digests, usage_events

  5. Row Level Security
     NOT configured in the app — the API handles auth at the application
     level (all DB queries are scoped to user_id). RLS is optional.
     If you want RLS, you'd need to add policies per table.

  6. Storage buckets
     NOT needed — PDFs go to Cloudflare R2, not Supabase Storage.
